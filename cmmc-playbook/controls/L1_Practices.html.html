CMMC Level 1 Practices

TANO = Trust • Accuracy • Neutrality • Organization • Transparency

This page summarizes the core CMMC Level 1 (Foundational) practices in plain language, from a small-business / lab environment perspective.

Access Control (AC)

AC.L1-3.1.1 – Authorized Access

Plain language: Only people who are supposed to use a system should have an account.

What this looks like: unique user accounts, no shared logins, disabled old/stale accounts.

Example evidence: user list export, account provisioning procedure, HR offboarding checklist.

AC.L1-3.1.2 – Limit System Access

Plain language: Give people only the minimum access they need for their job.

What this looks like: role-based access groups (e.g., “Finance”, “IT Admin”, “Student Worker”).

Example evidence: group membership report, access request forms, approval workflow notes.

AC.L1-3.1.20 – External Connections

Plain language: Control and approve external connections like VPN, Wi-Fi, and cloud apps.

Example evidence: VPN configuration, Wi-Fi PSK/MFA policy, list of approved SaaS apps.

Identification & Authentication (IA)

IA.L1-3.5.1 – Identify & Authenticate Users

Plain language: Users must log in with an ID and password (no anonymous access).

Example evidence: login screenshots, local/domain policy settings, password policy document.

IA.L1-3.5.2 – Use Multifactor Authentication

Plain language: Important and remote access accounts must use MFA (password + extra factor).

Example evidence: MFA configuration screenshot, VPN/SSO settings, MFA enrollment report.

System & Information Integrity (SI), Media (MP), Physical (PE), Secure Communications (SC)

SI.L1-3.14.1 – Patch & Update Systems

Plain language: Keep operating systems and software updated.

Example evidence: patch schedule, update screenshots, WSUS/Intune/SCCM reports.

MP.L1-3.8.3 – Media Sanitization

Plain language: Wipe or destroy drives, USBs, and devices before reuse or disposal.

PE.L1-3.10.1 – Physical Access Control

Plain language: Control who can physically access servers, network gear, and workstations.

SC.L1-3.13.1 – Boundary Protection

Plain language: Use a firewall and avoid unnecessary open inbound ports.
